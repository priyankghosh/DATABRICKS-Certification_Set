Describe the medallion architecture
-It's a sequential data organisation and pipeline system of progressively cleaner data
-consists of three layers: bronze, silver, and gold:
-The bronze layer contains unvalidated data in its raw state
-The silver layer represents a validated, enriched version of the data that can be trusted for downstream analytics.
-The gold layer contains highly refined and aggregated data that powers analytics, machine learning, and production applications.


Why is the gold layer as the most common layer for data analysts using Databricks SQL?
-It contains highly refined and aggregated data that power analytics, machine learning and production applications
-Data shared with a customer would rarely be stored outside this level.
-Because aggregations, joins, and filtering are handled before data is written to the gold layer, users should see low latency query performance on data in gold tables

Describe persistence and scope of tables on Databricks
There are different tables based on what's required:
1. Global tables are available across all clusters in a workspace and can be accessed by all users with the appropriate permissions
2. Cluster-scoped tables are available only within a specific cluster and are not visible to other clusters or users
3. Notebook-scoped tables are available only within a specific notebook and are not visible to other notebooks or users

Persisting tables in a storage format allows them to be stored on disk and accessed more efficiently, which can improve query performance and reduce query latency.

Unmanaged tables:
-Not managed by DB platform

Describe the difference between a view and a temp view
VIEW:
-a logical representation of data stored in a table or other view
-does not store data, but provides a way to access and query underlying data
-since view doesn't store data, it's persisted in underlying tables. changes to the table will be reflected in the view

TEMP VIEW:
-view that is not persisted to disk, only available during the session that created it
-any changes to the underlying data will not reflected in the temporary view

How can you optimise performance using higher-order Spark SQL functions?
Higher order functions are functions that take other functions as input parameters. They optimise performance by:
1. Can simplify code (write more readable code by reducing amount of logic and structure (ex: array_contains ( ) to check for a specific value))
2. Improve performance (can reduce the amount of data that needs to be filtered (ex: filter() can apply a predicate to a dataset))
3. Use with User Defined Functions to create more complex and powerful transformations (ex: transform ( ) to apply UDFs to each element of an array)
4. Use with window functions to perform complex and powerful aggregations and calculations (ex: can use collect_list () to collect column into an array, then percentile_approx() to calculate approximate percentile of the values)

Identify what happens if a refresh rate is less than the Warehouse's "Auto Stop"
If the refresh rate is less than the Auto Stop, the warehouse could be stopped between refreshes. This pauses all activity.

To avoid this, make sure the refresh rate is longer than Auto Stop, or make sure the warehouse never automatically stops.

CONTINUOUS analyses data that take on variations within a range. Examples include mean, standard deviation and correlation coefficient.


Describe descriptive statistics
Statistics used to describe and summarise a dataset. This includes central tendency, variability and distribution.


Describe key moments of statistical distributions
Numerical measurements that describe the shape, centre and spread of the distribution.


Compare and contrast key statistical measures
1. MEAN AND MEDIAN: Mean is average, median is the value in the middle. Mean is sensitive to outliers, median is not.

2. VARIANCE AND STANDARD DEVIATION: Both measures of spread, but SD is the square root of the variance. SD is easier to interpret as it's the same units as original data.

3. SKEWNESS AND KURTOSIS: Both measures of the shape of distribution. Skewness measures degree of asymmetry, kurtosis measures peakedness. Positive skewness means longer right tail, negative longer left tail. High kurtosis is high peaks and heavy tails, low kurtosis low peaks and light tails.

4. RANGE AND INTERQUARTILE RANGE: Measures of spread of data, with IQ range less sensitive to outliers. Range is difference between min and max range, IQ range is between 25th and 75th


Describe data enhancement as a common analytics application
Data enhancement is the process of improving the quality and usefulness of a dataset by adding or enriching with additional information. This includes a variety of techniques such as data cleaning, data integrations, data normalisation and data augmentation.


Describe the different techniques of data enhancement
DATA CLEANING: Identify and correct errors or inconsistencies in the data, such as missing values or incorrect data. Can be done manually or with automated tools.

DATA INTEGRATIONS: Combine data from multiple sources for a more comprehensive dataset. Data mapping and data transformation can be used to integrate the data.

DATA NORMALISATION: Organising the data in a consistent and standardised way. Can involve converting data into a common format.

DATA AUGMENTATION: Adding new data to a dataset to a dataset to improve quality or usefulness. Can involve collecting new data, or using ML for synthetic data.


Describe the blending of data between two source applications
Blending of data can be done using techniques such as:

DATA MAPPING: Identify common fields between two sources and map them to create a unified schema.

DATA TRANSFORMATION: Converting the data into a common format or structure so it can be combined.

DATA AGGREGATION: Summarise the data from each source and combine it into one dataset.


Perform last-mile ETL as project-specific data enhancement
Last-mile ETL refers to the final stage of the ETL (extract, transform, load) process, where data is transformed and enriched to meet the specific needs of a particular project or use case.
